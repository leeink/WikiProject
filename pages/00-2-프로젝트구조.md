## 우리가 작업할 코틀린(또는 자바) 프로젝트 구조

코틀린 프로젝트 구조는 흔히 다음과 같다.

```
MyApp/
├── gradle/wrapper/
│   ├── gradle-wrapper.jar
│   └── gradle-wrapper.properties
├── app/ Android 모듈
│   ├── src/main/
│   │   ├── java(kotlin)/com/example/wikiapp/
│   │   │   ├── MainActivity.kt 진입점
│   │   │   ├── ui/
│   │   │   │   ├── theme/
│   │   │   │         ├── Color.kt
│   │   │   │         ├── Theme.kt
│   │   │   │         ├── Type.kt
│   │   ├── AndroidManifest.xml 필수
│   │   └── res/
│   │       ├── values/
│   │             ├── colors.xml
│   │             ├── strings.xml
│   │             ├── themes.xml
│   └── build.gradle.kts 앱 빌드설정
│   └── src/test/
│       ├── java(kotlin)/com/example/wikiapp/
│            ├── ExampleUniTest 테스트 코드 작성
├── build.gradle.kts 루트 빌드설정
├── settings.gradle.kts
├── gradlew / gradlew.bat
├── gradle.properties
└── .gitignore
```

먼저 `src` 폴더는 프로젝트에서 사용하는 소스 파일이 모두 모여 있는
폴더다. `src`는 source의 약어다.

우리가 앞 장에서 별생각 없이 지나쳤던 부분들을 여기서 설명할 것이다.

```kotlin
package com.example.wikiapp
```

이런 형태를 소스 파일 최상단에서 볼 수 있었을 것이다.

`src/main/java(또는 kotlin)/com/example/wikiapp`이
위와 같은 형태로 정의된 것이다. 패키지는 관련된 파일들을 묶는
논리적인 그룹이다. 카테고리별로 묶는다고 이해하면 된다.

- `com`: 도메인 최상위 레벨
- `example`: 보통 조직(기업 또는 팀)의 이름이 기재된다.
- `wikiapp`: 앱이나 프로젝트 이름

보통 도메인이라고 한다면 `www.google.com` 이런 식인데
자바와 코틀린 프로젝트에서는 거꾸로 쓴다.
도메인은 전 세계에서 유일하게 소유권이 보장되어서, 이를 기반으로
패키지 이름을 만들면 전 세계 어떤 라이브러리와도 충돌하지 않는다.

Android 프로젝트의 경우라면 Play Store에서 앱을 식별하는 고유한
키가 된다.

`src/main/res/`는 앱에서 사용할 이미지 같은 에셋들을
모아두는 곳이다. 여태까지 아이콘은 기본 제공되는 아이콘만 사용했으나
아이콘들도 따로 만들어서 이 폴더에 저장하고 써도 된다.
이 폴더에 저장되는 파일 형식은 `.xml`이다.

- `values/colors.xml`: 프로젝트에서 사용할 색상을 미리 정의한다. `ui/theme/Color.kt`를 사용하면 되어서 사용할 필요 없다.
- `values/strings.xml`: 문자열을 사용할 때 변수 참조 형식으로 사용할 수 있게 한다. 이 파일을 사용하는 이유는 다국어 지원을 위해서이다.
- `values/themes.xml`: 프로젝트에서 사용할 테마를 미리 정의한다. `ui/theme/Theme.kt`를 사용하면 되어서 사용할 필요 없다.

`src` 폴더 외부에 있는 파일들은 주로 빌드 설정을 위한
파일들이다. 빌드라는 것은 우리가 작성한 코드들을 실제 앱으로
만들어 내는 것이다. Java나 Kotlin 빌드 설정이 너무 장황해서
이러한 것들을 사용한다.

- `settings.gradle.kts`: 프로젝트 구조를 정의하기 위함.
- `build.gradle.kts`: 빌드에 필요한 설정을 하기 위함. 라이브러리 추가를 이 파일에서 함.
- `gradle`이 붙은 파일들: 빌드 설정을 Gradle이라는 것을 사용해서 하는데 그에 필요한 파일들이다.
