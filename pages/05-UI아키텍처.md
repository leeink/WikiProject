이번 장에서는 Jetpack Compose의 UI아키텍처에 대해 알아볼 것이다.
이론적인 설명이 더 많아지므로 지루할 수 있다.

다음으로 넘어가기에 알아둬야 할 것들을 설명하겠다.

### 컴포저블, 컴포지션, 리컴포지션

- **컴포저블(Composable)**: Compose의 기본 빌딩 블록이다. 화면에 보여지는
모든 것들은 컴포저블인 것이다.
- **컴포지션(Composition)**: Composable 함수들이 실행되고 그 결과로 만들어지는
UI 트리이다. Compose는 UI를 트리구조로 관리한다.
- **리컴포지션(Re-Composition)**: 상태가 변경되면 Composable함수들이
선택적으로 재실행되는 것이다. 무조건 모든 Composable함수들을 다시 실행하지
않는다.

### Jetpack Compose의 UI는 변경될 수 없다.
이 말의 의미는 단순하다. 이미 코드로 작성을 완료한 UI 설계가 변경될 일이 없다는 뜻이다.
이해가 안될 수도 있는데 예시를 하나 들어보겠다.

```kotlin
@Composable
fun SampleComposable() {
    var isShow: Boolean by remember { mutableStateOf(false) }
    
    Button(onClick = {
        isShow = !isShow
    }){
        Text(text = "Show Toggle")
    }
    
    if(isShow){
        Text(
            text = "Show!"
        )
    }
}
```

위 경우에는 UI가 변경되는가? 라고 한다면 변경되지 않는다.
'아니 분명히 버튼 누를때마다 보였다 안보였다 하는데 변경되는 거 아니야?' 라고 할 수 있는데
엄밀히 말해서 UI가 변경되지 않는다는 말은 설계가 변경되지 않는다는 것이다.
위에서 버튼을 눌렀다고 해서 코드(설계)가 변경 되지 않는다.  

이러한 특성을 **불변성**이라고 한다.
