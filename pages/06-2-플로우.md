### 1. 차가운 흐름

Flow는 비동기를 사용하면서 데이터가 발생할 때마다 데이터가 발생할 때마다 지속적으로 데이터를 공급할 수가 있다.

보통 Flow는 Cold하다고 표현하는데 그 이유는 다음과 같다.

```kotlin
fun flowExample(): Flow<String> = flow {
    for (i in 1..5) {
        delay(300.milliseconds)
        emit("Flow data $i")
    }
}

fun main() = runBlocking<Unit> {
    launch {
        for (k in 1..5) {
            println("${k}번째 Data")
            delay(300.milliseconds)
        }
    }
    flowExample()
}


/* result
1번째 Data
2번째 Data
3번째 Data
4번째 Data
5번째 Data
*/
```

위는 Flow로 데이터를 출력하려는 예시인데 분명 flowExample 함수를 호출 했는데도
Flow Data구문이 출력되질 않는다. flowExample함수의 emit이 있는데
emit이 데이터를 보내는 함수이다. 무슨 문제로 데이터가 출력 되지 않을까?
Flow가 차갑다는 이유가 바로 이것인데, 별도로 함수를 호출하지 않는 이상 
데이터는 보내지지 않는다. 그럼 아래 예시를 보자.

```kotlin
fun flowExample(): Flow<String> = flow {
    for (i in 1..5) {
        delay(300.milliseconds)
        emit("Flow data $i")
    }
}

fun main() = runBlocking<Unit> {
    launch {
        for (k in 1..5) {
            println("${k}번째 Data")
            delay(300.milliseconds)
        }
    }
    flowExample().collect { value -> println(value) }
}
/* result
1번째 Data
Flow data 1
2번째 Data
Flow data 2
3번째 Data
Flow data 3
4번째 Data
Flow data 4
5번째 Data
Flow data 5
*/
```

Flow객체의 collect함수를 호출하고 나서야 데이터가 정상적으로 출력이 된다.
collect함수를 호출하는 것을 구독 행위로 본다면 구독을 한 뒤 알림을 실시간으로 계속
받고 있는 모습이라고 볼 수 있다. 실시간으로 데이터를 전송하지만 collect함수를 호출해야만
전송이 시작된다. 

그래서 아래와 같은 것이 가능하다.

```kotlin
fun flowExample(): Flow<String> = flow {
    for (i in 1..5) {
        delay(300.milliseconds)
        emit("Flow data $i")
    }
}

fun main() = runBlocking<Unit> {
    println("Main Started")

    val f = flowExample()

    println("Collect 호출")
    f.collect { value -> println(value) }

    println("Collect 호출")
    f.collect { value -> println(value) }
}
/* result
Main Started
Collect 호출
Flow data 1
Flow data 2
Flow data 3
Flow data 4
Flow data 5
Collect 호출
Flow data 1
Flow data 2
Flow data 3
Flow data 4
Flow data 5
*/
```

실시간으로 데이터를 전송받는 데 여러 종류의 데이터를 실시간으로 받아야 할 수도 있다.
그럼 아래와 같이 하면 된다.


```kotlin
fun flowExample(): Flow<String> = flow {
    for (i in 1..5) {
        delay(300.milliseconds)
        emit("Flow data $i")
    }
}

fun main() = runBlocking<Unit> {
    println("Main Started")

    val f = flowExample()

    println("Collect 호출")

    launch {
        f.collect { value -> println("Video data collect: $value") }
    }

    launch {
        f.collect { value -> println("Sound data collect: $value") }
    }
}
/* result
Main Started
Collect 호출
Video data collect: Flow data 1
Sound data collect: Flow data 1
Video data collect: Flow data 2
Sound data collect: Flow data 2
Video data collect: Flow data 3
Sound data collect: Flow data 3
Video data collect: Flow data 4
Sound data collect: Flow data 4
Video data collect: Flow data 5
Sound data collect: Flow data 5
*/
```

Flow도 코루틴처럼 취소 할 수 있다. 취소할 때 간편하게 취소하는 방법은 시간제한을
두는 것이다.

```kotlin
fun flowExample(): Flow<String> = flow {
    for (i in 1..5) {
        delay(300.milliseconds)
        emit("Flow data $i")
    }
}

fun main() = runBlocking<Unit> {
    println("플로우가 취소됨.")
    withTimeoutOrNull(900.milliseconds){
        flowExample().collect { value -> println("Flow collect: $value") }
    }
}

/* result
플로우가 취소됨.
Flow collect: Flow data 1
Flow collect: Flow data 2
*/
```

취소할 때 안전하게 취소하도록 시간제한을 두고 시간이 지나면 에러를 반환하는 것이 아닌
null을 반환하도록 한 것이다.  

Flow를 만들 때 flow빌더 함수를 썼는데 간편하게 만들 수도 있다.

```kotlin
fun main() = runBlocking<Unit> {
    (1..3).asFlow().collect { value -> println("${value}번째 data") }
}

/* result
1번째 data
2번째 data
3번째 data
*/
```

### 2. 연산

데이터를 가져올 때 추가적인 연산이나 변환이 필요한 경우가 있을 것이다.
Flow는 이러한 연산을 중간에 할 수 있게 지원한다.

```kotlin
fun flowExample(): Flow<String> = flow {
    for (i in 1..5) {
        delay(300.milliseconds)
        emit("Flow data $i")
    }
}

fun main() = runBlocking<Unit> {
    flowExample()
        .map{value -> "Data collect: $value" }
        .collect { value -> println(value) }
}
/* result
Data collect: Flow data 1
Data collect: Flow data 2
Data collect: Flow data 3
Data collect: Flow data 4
Data collect: Flow data 5
*/
```

collect함수가 호출 되기 전 중간에 연산이 가해지고 난 후 데이터가 수집된다.
꼭 map이 아니라 filter나 다른 함수도 가능하다.

```kotlin
fun flowExample(): Flow<Int> = flow {
    for (i in 1..5) {
        delay(300.milliseconds)
        emit(i)
    }
}

fun main() = runBlocking<Unit> {
    flowExample()
        .filter{ it % 2 == 0 }
        .collect { value -> println("Filtered Data = $value") }
}

/* result
Filtered Data = 2
Filtered Data = 4
*/
```

중간에 연산이 들어가면 filter같은 것들을 중간 연산자라고 하는데 마지막에 연산이 가해지는
종단 연산자도 있다. 종단 연산자는 이후 나오는 절에서 예시로 살펴볼 수 있다.


### 3. 어디서 실행 할 것인가


코루틴처럼 withContext로 수집을 하는 곳에서 실행환경을 변경할 수 있다. Flow는 데이터를
발생 시키는 곳(emit호출)을 생산자, 수집 하는 곳(collect호출)을 소비자라고 하는데
생산자쪽에서 실행환경을 변경할 수 있다.

```kotlin
fun flowExample(): Flow<Int> = flow {
    for (i in 1..5) {
        println("Supplier Thread: ${Thread.currentThread().name}: $i")
        delay(300.milliseconds)
        emit(i)
    }
}.flowOn(Dispatchers.Default)

fun main() = runBlocking<Unit> {
    flowExample()
        .collect { value -> println("Consumer Thread: ${Thread.currentThread().name}: $value") }
}
```

flowOn함수로 실행환경을 변경할 수 있다.
