### 1. 로딩 중과 에러

이제 HomeScreen에 여러 정보를 표시할 것이다. 다음과 같은 시나리오로 진행한다.

1. 홈 스크린 진입
2. 데이터를 가져오는 데 약간의 시간이 걸림
3. 데이터를 가져오는 중에는 가져오고 있다는 메시지를 화면에 표시
4. 가져오는 데 성공하면 리스트가 보일 것이고 실패하면 실패했다는 메시지를 화면에 표시

```kotlin
data class Article(
    val title: String = "",
    val author: String = "",
    val content: String = "",
)

data class ArticleUiState(
    val articles: List<Article> = listOf(),
    val isLoading: Boolean = false,
    val isError: Boolean = false,
    val message: String = "Fetching Articles...",
)

class ArticleViewModel : ViewModel() {
    private val _articles = MutableStateFlow(ArticleUiState())
    val articles: StateFlow<ArticleUiState> = _articles.asStateFlow()
}
```

(fakeArticles 리스트는 생략 — 이전과 동일)

HomeScreen에는 기사(Article)를 리스트로 나열할 것이다.
그래서 `ArticleUiState`에는 List로 Article들을 저장할 것이다. `isLoading`과 `message`는 데이터를
가져오고 있는지에 대한 여부와 그것을 화면에 표시할 것들이다. `isError`는 에러가 발생했는지에 대한 여부이다.

다음은 ViewModel을 마저 작업할 것이다.

```kotlin
class ArticleViewModel : ViewModel() {
    private val _articles = MutableStateFlow(ArticleUiState())
    val articles: StateFlow<ArticleUiState> = _articles.asStateFlow()

    init {
        fetchArticles()
    }

    fun fetchArticles() {
        viewModelScope.launch(Dispatchers.IO) {
            _articles.update { it.copy(isLoading = true, message = "Fetching Articles...") }
            try {
                val articles = fakeArticles
                _articles.update { it.copy(articles = articles, isLoading = false, isError = false, message = "Done") }
            } catch (e: Exception) {
                _articles.update { it.copy(isLoading = false, isError = true, message = e.message ?: "Not Found Error...") }
            }
        }
    }
}
```

`fetchArticles` 함수가 데이터를 가져올 것이다. 여기서 `viewModelScope.launch`가 보일 것이다.
6장에서 본 비동기 구문과 매우 유사하다.

`viewModelScope`는 ViewModel에서 제공되는 코루틴 스코프이다. ViewModel이
살아있는 동안에만 코루틴이 실행된다.

이렇게 하고 실행해보면 데이터가 화면에 바로 나타난다.
하지만 우리가 시뮬레이션해보고 싶은 것은 데이터를 가져오는 데 어느 정도 시간이
걸리는 경우이다. `ArticleScreen`에서 `ArticleList()` 부분을 지우고 주석 처리된
부분의 주석을 해제하자. 그리고 `fetchArticles` 함수에 `delay`를 추가한다.

```kotlin
fun fetchArticles() {
    viewModelScope.launch(Dispatchers.IO) {
        _articles.update { it.copy(isLoading = true, message = "Fetching Articles...") }
        try {
            delay(5000.milliseconds)
            val articles = fakeArticles
            _articles.update { it.copy(articles = articles, isLoading = false, isError = false, message = "Done") }
        } catch (e: Exception) {
            _articles.update { it.copy(isLoading = false, isError = true, message = e.message ?: "Not Found Error...") }
        }
    }
}
```

이제 실행하면 CircularProgressIndicator가 돌면서 로딩 상태가 되고 데이터를 다 가져오면
그제서야 리스트가 보일 것이다.

그다음은 에러 화면을 보는 시뮬레이션을 할 것이다. `fetchArticles` 함수에
간단하게 오류를 던지자.

```kotlin
fun fetchArticles() {
    viewModelScope.launch(Dispatchers.IO) {
        _articles.update { it.copy(isLoading = true, message = "Fetching Articles...") }
        try {
            delay(5000.milliseconds)
            throw Exception()
            val articles = fakeArticles
            _articles.update { it.copy(articles = articles, isLoading = false, isError = false, message = "Done") }
        } catch (e: Exception) {
            _articles.update { it.copy(isLoading = false, isError = true, message = e.message ?: "Not Found Error...") }
        }
    }
}
```

그러면 우리가 만들었던 `ErrorScreen`이 보일 것이다.

### 2. 데이터 가져올 때 스레드 전환하기

```kotlin
class ArticleViewModel : ViewModel() {
    private val _articles = MutableStateFlow(ArticleUiState())
    val articles: StateFlow<ArticleUiState> = _articles.asStateFlow()

    init {
        fetchArticles()
    }

    fun fetchArticles() {
        viewModelScope.launch(Dispatchers.IO) {
            _articles.update { it.copy(isLoading = true, message = "Fetching Articles...") }
            delay(5000.milliseconds)
            try {
                val articles = fakeArticles
                _articles.update { it.copy(articles = articles, isLoading = false, isError = false, message = "Done") }
            } catch (e: Exception) {
                _articles.update { it.copy(isLoading = false, isError = true, message = e.message ?: "Not Found Error...") }
            }
        }
    }
}
```

위 예시에서는 함수를 통째로 IO 스레드로 전환해서 했지만 그렇게 하지 않고
중간에 `withContext`로 스레드를 전환해 가며 할 수 있다.

```kotlin
fun fetchArticles() {
    viewModelScope.launch {
        _articles.update { it.copy(isLoading = true, message = "Fetching Articles...") }

        withContext(Dispatchers.IO) {
            delay(5000.milliseconds)
            try {
                val articles = fakeArticles
                _articles.update { it.copy(articles = articles, isLoading = false, isError = false, message = "Done") }
            } catch (e: Exception) {
                _articles.update { it.copy(isLoading = false, isError = true, message = e.message ?: "Not Found Error...") }
            }
        }
    }
}
```

그런데 이것도 좋지 않다고 볼 수 있다. UI 상태 변경은 Main 스레드에서
하는 게 좋은데 지금은 처음 로딩 진입점만 Main 스레드에서 하고
나머지는 IO 스레드에서 하고 있다. 한 번 더 범위를 좁혀보자.

```kotlin
fun fetchArticles() {
    viewModelScope.launch {
        _articles.update { it.copy(isLoading = true, message = "Fetching Articles...") }

        try {
            val articles = withContext(Dispatchers.IO) {
                delay(5000.milliseconds)
                fakeArticles
            }
            _articles.update { it.copy(articles = articles, isLoading = false, isError = false, message = "Done") }
        } catch (e: Exception) {
            _articles.update { it.copy(isLoading = false, isError = true, message = e.message ?: "Not Found Error...") }
        }
    }
}
```

이렇게 하면 데이터를 가져오는 것만 IO 스레드에서 하고 나머지 UI 상태
변경은 Main 스레드에서 하게 될 것이다.
